E-mail Templates

Introduction

The e-mail templates are the templates that control the content of the e-mail messages that will be sent to your users when they add an account, submit a link, etc. This document will give you an overview of the variables available in each of the e-mail templates.

Editing E-mail Templates

To edit the e-mail templates access the Templates > E-mail Templates function from the control panel menu. This will display a page where you can select which template you want to edit. Once you have selected the template you want to edit, press the Load Template button. You will be returned to the same page, but there will now be a three fields that you will need to fill in:
    Subject The full subject of the rejection e-mail message
    Text Body The text body of the rejection e-mail message. This message will be displayed for users that have text-only e-mail readers.
    HTML Body The HTML body of the rejection e-mail message. This message will be displayed for users that have HTML capable e-mail readers.
    TIP: Click on the icon to automatically convert the text body to HTML format.
When you are done making changes to the template, press the Save Template button to save your changes.

Templates and Their Variables

This section will describe all of the available templates and the variables you can use within each template.
    email-account-added.tpl

    This e-mail message is sent to the user after they have succesfully created or confirmed their account

      {$config} - The LinkX configuration settings; see the Common Variables section for details
      {$account} - The account data; see the Common Variables section for details
      {$user_fields} - The user defined fields for this account (array value)


    email-account-confirm.tpl

    This e-mail message is sent to the user after they have submitted their account information but need to confirm the submission through their e-mail address

      {$config} - The LinkX configuration settings; see the Common Variables section for details
      {$account} - The account data; see the Common Variables section for details
      {$user_fields} - The user defined fields for this account (array value)
      {$confirm_id} - The confirmation ID that must be supplied in the confirmation link to confirm this request


    email-account-forgot.tpl

    This e-mail message is sent when a user has requested a new password for their account. This is the confirmation e-mail which contains a link that they must click to confirm that they requested a new password.

      {$config} - The LinkX configuration settings; see the Common Variables section for details
      {$account} - The account data; see the Common Variables section for details
      {$confirm_id} - The confirmation ID that must be supplied in the confirmation link to confirm this request


    email-account-password.tpl

    This e-mail message will be sent to a user after they have confirmed that they requested a new password. It will contain their new password.

      {$config} - The LinkX configuration settings; see the Common Variables section for details
      {$account} - The account data; see the Common Variables section for details


    email-admin-comment.tpl

    This e-mail message will be sent to administrators who are configured to receive a notice when a new comment is added.

      {$config} - The LinkX configuration settings; see the Common Variables section for details
      {$errors} - If any errors were encountered, this value will contain them


    email-admin-link-add.tpl

    This e-mail message will be sent to administrators who are configured to receive a notice when a new link is added.

      {$config} - The LinkX configuration settings; see the Common Variables section for details
      {$status} - The status assigned to this link (one of unconfirmed, pending, or active)
      {$category} - The category the user has selected to add this link to
      {$user_fields} - The user defined link fields
      {$link} - The link data that the user has submitted; see the Common Variables section for details


    email-admin-link-edit.tpl

    This e-mail message will be sent to administrators who are configured to receive a notice when a link is edited.

      {$config} - The LinkX configuration settings; see the Common Variables section for details
      {$categories} - The categories this link is assigned to (array value)
      {$user_fields} - The user defined link fields
      {$link} - The link data that the user has submitted; see the Common Variables section for details


    email-link-added.tpl

    This message will be sent to the user after they have successfully submitted or confirmed their link

      {$config} - The LinkX configuration settings; see the Common Variables section for details
      {$status} - The status assigned to this link (one of unconfirmed, pending, or active)
      {$category} - The category the user has selected to add this link to
      {$user_fields} - The user defined link fields
      {$link} - The link data that the user has submitted; see the Common Variables section for details

      {$config} - The LinkX configuration settings; see the Common Variables section for details
      {$status} - The status assigned to this link (one of unconfirmed, pending, or active)
      {$category} - The category the user has selected to add this link to
      {$user_fields} - The user defined link fields
